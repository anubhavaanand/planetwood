Loading Screen

Full-screen overlay shown during game asset loading. Displays logo, progress bar, status text, and tip cycling. Covers all loading states including error recovery.

States

Initial (0%)
Mid-Progress (47%)
Near Complete (92%)
Complete (100%)
Error (corrupt)
Error (network)
Error (reconnecting)

API

MethodSignatureDescription
show()() => voidShows the loading screen with fade-in
hide()() => voidHides the loading screen with fade-out
setProgress(n)(n: number) => voidSets progress 0–100, updates bar and aria-valuenow
setSubProgress(n)(n: number) => voidSets sub-file progress 0–1 within the current step
setStatus(text)(text: string) => voidSets the status text shown below the progress bar
setError(err)(err: object) => voidShows error state with message and retry button
setTip(index)(index: number) => voidCycles to a specific tip by index
cycleTip()() => voidAdvances to the next tip (called on interval)
isOpengetter => booleanReturns whether the screen is currently visible

Tips Array

Cycles every 8 seconds during loading:

[ "Explore a living world with unique NPCs", "Chat with other players in real time", "Use emotes to express yourself", "Collect hidden treasures across the planet", "Customize your avatar appearance" ]

Events

EventDetailWhen
loading:retry{}User clicks retry button
loading:dismiss{}User clicks report / dismiss
Accessibility
  • role="progressbar" with aria-valuenow for progress
  • aria-live="polite" on status and tip text
  • aria-busy="true" on container while loading
  • Focus trapping during loading (cannot tab out)
  • prefers-reduced-motion: disables all transitions
  • Error state has role="alertdialog" with focus trap

Integration

// Create CSS2DRenderer nameplate for a player const div = document.createElement('div'); div.className = 'pw-nameplate'; div.innerHTML = '<span class="status-dot online"></span>' + '<span class="player-name">' + name + '</span>' + (isLocal ? '<span class="you-badge">You</span>' : ''); const label = new CSS2DObject(div); label.position.set(x, y + 2.5, z); // offset above avatar head nameplateGroup.add(label); // On each frame: project 3D position to screen via CSS2DRenderer // CSS2DRenderer automatically handles screen-space rendering
Note: Nameplates use CSS2DRenderer (not sprite-based) so text is always crisp regardless of camera distance. The renderer is separate from the WebGL renderer and renders on top.
Accessibility
  • role="status" on each nameplate
  • aria-label combining name and status: "BraveFox - online"
  • Interactive (clickable for chat context)
  • pointer-events: auto enables clicking through CSS2DRenderer layer
