Technical Spec — T020

Nameplate CSS2DRenderer

How player nameplates render in 3D space using Three.js CSS2DRenderer. Bridges the HTML/CSS nameplate component with in-world 3D rendering. Guides chat bubble rendering (T020) and nameplate positioning.

02 — CSS2DRenderer Setup

Integration Guide

CSS2DRenderer creates a second DOM layer on top of the WebGL canvas. HTML elements positioned via CSS2DObject appear at 3D world coordinates but render at fixed pixel size — no perspective scaling, no lighting.

Import (Three.js r185)

JavaScript 
import { CSS2DRenderer, CSS2DObject } from 'three/examples/jsm/renderers/CSS2DRenderer.js';

Renderer Initialization

JavaScript 
// Create CSS2DRenderer — must render on top of WebGL
const labelRenderer = new CSS2DRenderer();
labelRenderer.setSize(window.innerWidth, window.innerHeight);
labelRenderer.domElement.style.position = 'absolute';
labelRenderer.domElement.style.top = '0';
labelRenderer.domElement.style.left = '0';
labelRenderer.domElement.style.pointerEvents = 'none'; // click-through to game
document.getElementById('game-container').appendChild(labelRenderer.domElement);

// In animate() loop — must render BOTH renderers
function animate() {
  requestAnimationFrame(animate);
  renderer.render(scene, camera);      // WebGL
  labelRenderer.render(scene, camera); // HTML overlays
}
Key constraint: pointerEvents: none is set on the container. Individual nameplate elements set pointer-events: auto when they need click interaction.
03 — Nameplate Factory

createNameplate()

Factory function that builds a DOM element matching the Planetwood nameplate design, wraps it in a CSS2DObject, and attaches it to an avatar's head bone position.

JavaScript 
function createNameplate(player) {
  const div = document.createElement('div');
  div.className = 'pw-nameplate';
  div.id = `nameplate-${player.id}`;
  div.setAttribute('role', 'status');
  div.setAttribute('aria-label', `${player.name} - ${player.status}`);

  // Status dot
  const dot = document.createElement('div');
  dot.className = `status-dot ${player.status}`;
  div.appendChild(dot);

  // Player name
  const name = document.createElement('span');
  name.className = 'player-name';
  name.textContent = player.displayName;
  div.appendChild(name);

  // "You" badge — only for local player
  if (player.isLocal) {
    const badge = document.createElement('span');
    badge.className = 'you-badge';
    badge.textContent = 'You';
    div.appendChild(badge);
  }

  // Wrap in CSS2DObject and attach to head bone
  const label = new CSS2DObject(div);
  label.position.set(0, headHeight, 0);

  return label;
}

Billboard Behavior

CSS2DRenderer automatically billboards elements — they always face the camera. No rotation interpolation needed. For optional depth perception, scale the nameplate based on camera distance:

JavaScript 
const dist = camera.position.distanceTo(label.position);
const scale = Math.max(0.5, Math.min(1.5, 10 / dist));
div.style.transform = `scale(${scale})`;
05 — Z-Order & Layering

Depth Sorting

CSS2DRenderer places elements in document order. Nameplates of closer players should appear on top. Sort before each render by distance to camera.

JavaScript 
function sortNameplates(camera) {
  const nameplates = playerNameplateGroup.children;
  nameplates.sort((a, b) => {
    const distA = camera.position.distanceTo(a.position);
    const distB = camera.position.distanceTo(b.position);
    return distB - distA; // farthest first, closest last (on top)
  });
  nameplates.forEach((np, i) => {
    np.element.style.zIndex = i;
  });
}

06 — Chat Bubble Integration (T020)

Chat Bubble Rendering

Chat bubbles are CSS2DObjects positioned above nameplates. They appear when a player sends a message and auto-remove after 5 seconds.

Factory Function

JavaScript 
function createChatBubble(senderId, senderName, text) {
  const div = document.createElement('div');
  div.className = 'pw-chat-bubble';
  div.innerHTML = `
    <div class="bubble-name">${escapeHtml(senderName)}</div>
    <div class="bubble-text">${escapeHtml(text)}</div>
  `;

  const bubble = new CSS2DObject(div);
  // Position above nameplate
  bubble.position.set(0, headHeight + 0.8, 0);

  return bubble;
}

07 — State Management

NameplateManager

Manages the lifecycle of all nameplates. Handles add, remove, position updates, and status changes. Owns the THREE.Group that holds all CSS2DObjects.

JavaScript 
class NameplateManager {
  constructor(scene) {
    this.nameplates = new Map(); // playerId → CSS2DObject
    this.group = new THREE.Group();
    this.group.name = 'nameplates';
    scene.add(this.group);
  }

  addPlayer(player) {
    const label = createNameplate(player);
    this.nameplates.set(player.id, label);
    this.group.add(label);
  }

  removePlayer(playerId) {
    const label = this.nameplates.get(playerId);
    if (label) {
      this.group.remove(label);
      label.element.remove(); // clean up DOM
      this.nameplates.delete(playerId);
    }
  }

  updatePosition(playerId, position, headHeight) {
    const label = this.nameplates.get(playerId);
    if (label) {
      label.position.copy(position);
      label.position.y += headHeight;
    }
  }

  updateStatus(playerId, status) {
    const label = this.nameplates.get(playerId);
    if (label) {
      const dot = label.element.querySelector('.status-dot');
      dot.className = `status-dot ${status}`;
    }
  }

  sortByDistance(camera) {
    const items = Array.from(this.nameplates.values());
    items.sort((a, b) => {
      const da = camera.position.distanceTo(a.position);
      const db = camera.position.distanceTo(b.position);
      return db - da;
    });
    items.forEach((np, i) => { np.element.style.zIndex = i; });
  }
}

ChatBubbleManager

JavaScript 
class ChatBubbleManager {
  constructor(scene) {
    this.bubbles = new Map(); // playerId → { bubble, timeoutId }
    this.group = new THREE.Group();
    this.group.name = 'chat-bubbles';
    scene.add(this.group);
  }

  showMessage(senderId, senderName, text, position, headHeight) {
    this.removeBubble(senderId);

    const bubble = createChatBubble(senderId, senderName, text);
    bubble.position.set(position.x, position.y + headHeight + 0.8, position.z);
    this.group.add(bubble);

    // Trigger fade-in after DOM attachment
    requestAnimationFrame(() => {
      bubble.element.classList.add('visible');
    });

    // Auto-remove after 5 seconds
    const timeoutId = setTimeout(() => this.removeBubble(senderId), 5000);
    this.bubbles.set(senderId, { bubble, timeoutId });
  }

  removeBubble(playerId) {
    const existing = this.bubbles.get(playerId);
    if (existing) {
      clearTimeout(existing.timeoutId);
      this.group.remove(existing.bubble);
      existing.bubble.element.remove();
      this.bubbles.delete(playerId);
    }
  }
}
08 — Performance

Considerations

ConcernDetailMitigation
DOM count CSS2DRenderer creates real DOM nodes 10 nameplates + 10 bubbles = ~20 nodes. Fine for Planetwood's 10-player cap.
Memory Orphaned elements accumulate Call removePlayer() on disconnect. DOM nodes + event listeners freed.
Layout thrashing Reading offsetHeight forces reflow Batch DOM reads/writes via requestAnimationFrame. Use transform not top/left.
reducedMotion Fade animations cause motion sickness @media (prefers-reduced-motion: reduce) — skip fades, show/hide instantly.
renderOrder Overlay draw order relative to WebGL label.renderOrder = 1 ensures always on top of meshes.
CSS 
/* Respect reduced motion preference */
@media (prefers-reduced-motion: reduce) {
  .pw-chat-bubble {
    transition: none;
  }
  .pw-chat-bubble.visible {
    opacity: 1;
    transform: none;
  }
}
09 — Troubleshooting

Common Issues

Nameplate not showing

Cause: labelRenderer.render() not in animate loop
Fix: Add labelRenderer.render(scene, camera) to your animate() function

Nameplate behind WebGL

Cause: CSS2DRenderer DOM element z-index too low
Fix: Set labelRenderer.domElement.style.zIndex = '1'

Click events blocked

Cause: pointer-events: none on the container blocks all interaction
Fix: Set pointer-events: auto on individual nameplate elements

Nameplate jitters

Cause: Layout thrashing from forced reflow on every frame
Fix: Batch DOM reads/writes. Use transform instead of top/left

Chat bubble cuts off

Cause: overflow: hidden on CSS2DRenderer parent
Fix: Ensure parent has overflow: visible

Wrong position in VR

Cause: CSS2DRenderer is not VR/XR-aware
Fix: Hide nameplates in VR mode, use sprite text instead
